## 🧠 サービス概要

学習のつまずきを減らし、次の一歩へ進みやすくしてくれる  
**“学習ナレッジベース × アドバイスツール”**です。

本アプリは、学習で頻出するテーマを体系的に整理したカテゴリ群と、  
学習者のリアルな悩みを共有・解決できる「みんなのお悩み解決」カテゴリで構成されています。

アルゴリズム、Rails、SQL、設計、開発、エラーなどの学習カテゴリに加え、  
不安や継続を支える「モチベ」カテゴリも用意しています。

---

## 💡 このサービスへの思い・作りたい理由

自分自身が学習中に何度もエラーや理解不足に苦しみ、  
「今の状況に合ったアドバイスが欲しい」と感じる場面が多くありました。

AIに質問できる環境はありますが、  
**実際に同じ状況を経験した人の知見が最も役に立つ**と感じました。

そこで、学習者のリアルなつまずきと解決方法を整理し、  
次の学習者がすぐに活用できる形にしたいと考え、このサービスを開発しました。

## 🖼️ サービスの利用イメージ

1. 黒板UIからカテゴリを選択  
2. アドバイスボタンを選択  
3. フクちゃんがアドバイスを表示  
4. お悩み投稿 → 管理者承認 → 掲示板へ反映  

👉 **学習者の経験がアプリを成長させる仕組み**

---

### 🎬 アドバイス閲覧

![アドバイス](./docs/advice.gif)

カテゴリ選択からアドバイス表示までの流れです。  
タイピング演出と全文表示機能により、楽しさと読みやすさの両立を意識しています。

---

### 🌿 背景切り替え

![背景変更](./docs/background.gif)

アプリ設定から背景を変更可能です。  
デフォルトでは時間帯に応じて自動で切り替わる仕様にしています。

* 5:00〜10:59 → 春（朝）
* 11:00〜15:59 → 夏（昼）
* 16:00〜18:59 → 秋（夕方）
* 19:00〜4:59 → 冬（夜）

## 🎯 工夫した点

### ✔ キャラクターを活用したUX設計

ローディング画面では単なるスピナーではなく、  
フクちゃんが案内する演出を入れることで、  
**待機時間をストレスではなく体験として感じられるように設計しました。**

---

### ✔ ローディングUIの最適化

アニメーションの1サイクル（1秒）に合わせて表示時間を統一し、  
**途中で切れない自然な体験**を実現しました。

---

### ✔ 適用範囲の設計

すべての遷移に適用するのではなく、  
**ユーザーの意図が明確な遷移に限定**することでUXを維持しています。

---

### ✔ データ設計の工夫

* seedデータを冪等設計（seed_key）で管理  
* 不要データの自動削除  

👉 **本番運用を意識したデータ管理**

---

### ✔ インフラ構築（独自ドメイン対応）

* 独自ドメイン取得（お名前.com）  
* DNS設定（CNAME / Aレコード）  
* Renderとの接続  
* SSL証明書の自動発行（HTTPS化）  

👉 **本番環境構築まで一貫して対応**

## 🔍 既存サービスとの差別化

### ① 記事ストック型（Qiita等）

* 長文記事を読む必要がない  
👉 **短時間で必要な情報に到達可能**

---

### ② Q&A型（StackOverflow / teratail）

* 質問投稿不要  
👉 **質問が苦手でも利用できる**

---

### ③ 掲示板型

* 情報が流れない構造  
👉 **必要な情報をすぐに取得可能**

## 🧰 セットアップ手順

```bash
git clone https://github.com/fuku-0420/owl-forest-app.git
cd owl-forest-app

bundle install
yarn install

bin/rails db:create
bin/rails db:migrate
bin/rails db:seed

bin/rails server
```

👉 `http://localhost:3000` で起動
