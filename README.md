## プログラミング学習者向け掲示板  サービス概要

学習中によく出る疑問やエラーをカテゴリごとに整理し、迷わず調べられる学習サポートアプリです。  
黒板風の画面から悩みを選ぶだけで、フクちゃんが原因や対処ポイントを分かりやすく案内してくれます。  
学習のつまずきを減らし、次の一歩へ進みやすくしてくれる“学習ナレッジベース × アドバイスツール”です。

---

## このサービスへの思い・作りたい理由

自分自身が学習中に何度もエラーや理解不足に苦しみ、
「今の状況に合ったアドバイスが欲しい」と感じる場面が多くありました。

プログラミング学習では AI に質問したりできますが、AI の回答が間違っていることもあり、  
「実際に同じ状況を経験した受講生のリアルなエピソードが最も役に立つ」 と強く思いました。

そこで、受講生たちが経験したつまずきや解決方法を整理し、  
後輩が同じ壁にぶつかったときにすぐ役立つ“学習ナレッジ集”として形にすれば、  
確かな価値が生まれると考えました。

フクロウの“フクちゃん”が学習者に寄り添い、
受講生の経験をもとにアドバイスを返してくれるアプリを目指して、  
このサービスを作ることにしました。

## 🎯 ユーザー層について

### RUNTEQ受講生
- よくあるつまずきを知りたい
- 人に質問する前に自力で解決したい

### 独学のプログラミング初学者
- 今の自分と似た悩みを探したい
- 調べ方が分からず詰まることが多い

**理由**
プログラミング学習者は「悩みをうまく言葉にできない」「調べ方が分からず詰まる」ことが多いです。  
こうした背景から、悩みをカテゴリからすぐ探せて、フクちゃんがやさしく案内してくれるこのサービスは、  
人に質問する前に気軽に使える“最初の相談相手”として役立ちます。

## 🖼️ サービスの利用イメージ

1. トップ画面の黒板に「エラー」「モチベ」「設計」などカテゴリが並ぶ
2. カテゴリを押すと “アドバイスボタン” が表示される
3. ボタンを押すとフクちゃんからアドバイスが表示される
4. 受講生の知見は「アドバイス案募集」項目から追加され、
   採用されたものは新しい悩みボタンとして増えていく

  **学習者の経験がアプリ自体を成長させる仕組み**

## 🔍 既存サービスとの差別化

学習サポート系サービスには Qiita（記事ストック型）、StackOverflow / teratail（Q&A型）、  
スレッド掲示板型などがありますが、本サービスはそれらとは異なる  
**“悩みを素早く参照することに特化した構造”** を採用しています。

---

### ① 記事ストック型（Qiita等）との違い
- 長文記事を検索・読解する必要がない  
- 必要なアドバイスがカテゴリで整理されていて迷いにくい  
→ **知りたいポイントに短時間でアクセスできる**

### ② Q&A型（StackOverflow / teratail）との違い
- 質問投稿が不要（無回答リスクなし）  
- 質問文を考える負担がない  
→ **質問が苦手な学習者でも使いやすい**

### ③ 掲示板スレッド型との違い
- 情報が流れず、必要な悩みだけが常に見える  
- 過去ログを遡る手間が不要  
→ **効率よく目的の情報にたどり着ける**

---

## 🦉 このサービスならではの推しポイント（独自性）

### ✔ 学習者の実体験がベースの“信頼できる学習ナレッジ”
一般的な学習サイトでは古い情報や環境差による回答が混ざりがちですが、  
このサービスでは **学習者が実際につまずきやすいポイントに沿ってナレッジを整理**しています。  
「今まさに困っている問題」に自然とアクセスできます。

### ✔ フクちゃんによる“楽しく学べる心地よさ”
黒板UIとフクロウのキャラクターが、学習の雰囲気をやわらかくしてくれます。  
エラーで落ち込みやすい時期でも、**気持ちを切り替えて楽しく学べるデザイン**を重視しています。

### ✔ 経験がサービスを育てる“参加型の仕組み”
「アドバイス案募集」で集まった知見が新しいアドバイスボタンに反映され、  
**みんなでサービスを育てていく一体感**が生まれます。
